# B2 — `find_document_key` query method + MCP tool

**Priority:** P1 for v3.3 — turns B1's data into an agent-reachable surface
**Depends on:** [v3.3-b1-schema-and-i18n-json-parser.md](v3.3-b1-schema-and-i18n-json-parser.md)
**Plan reference:** `~/.claude/plans/9-v3.3-documents-surface.md`

## Goal

Add the query side of the documents surface. B1 puts entries in the
`documents` table; B2 makes them findable via MCP.

The expected usage from the v3.1 field test (F3):

> Agent needs to add a new i18n key `topbar.navigation.back: "Go back"`.
> Searches for `topbar.navigation` to see existing siblings, picks the
> right file, opens it with `read_focused`, adds the entry.

## Query

`internal/query/documents.go` (new):

```go
type DocumentHit struct {
    ID      int64  `json:"id"`
    Kind    string `json:"kind"`
    Path    string `json:"path"`
    Project string `json:"project,omitempty"`
    Key     string `json:"key"`
    Value   string `json:"value"`
    Line    int    `json:"line"`
}

func (r *Reader) FindDocumentKey(
    ctx context.Context,
    key, kind, project string,
    limit int,
) ([]DocumentHit, error)
```

SQL: substring match against `documents.key`, with `LEFT JOIN files`
+ `LEFT JOIN projects` to populate `Path` and `Project` (the v3.1.2
annotation convention). Optional `kind` and `project` filters.
Default `limit` 50.

Ordering: exact match > prefix match > substring match, then by
`key` length ascending. Same intent as `FindSymbol`'s ordering.

Each hit's `path` + `project` pass verbatim to `read_focused`.",

Parameters: `key` (required), `kind`, `project`, `limit`.
