# B1 — Documents schema + parser infrastructure + i18n JSON kind

**Priority:** P0 for v3.3 — everything else depends on this
**Plan reference:** `~/.claude/plans/9-v3.3-documents-surface.md`

## Goal

Stand up the "documents" surface as a parallel track to the symbol
graph. v3.3's job is to make i18n JSON keys (and later
`package.json` deps, `go.mod` requires) findable without dropping to
`Bash(grep)` — which the v3.1 field test showed is exactly what
agents do today.

After this ticket: indexing a repo with an i18n JSON file produces
flat key/value entries in a new `documents` table, the file itself is
in `files` (so `search_lexical` still works), and the existing symbol
graph is untouched.

## Schema

New migration `internal/index/migrations/0007_documents.sql`:

```sql
-- Add a document_kind column to files. NULL for code files; set to
-- the document kind (e.g. "i18n_json") for files indexed purely as
-- documents. Lets doctor and stats group document files cleanly
-- without overloading files.language.
ALTER TABLE files ADD COLUMN document_kind TEXT;
CREATE INDEX idx_files_document_kind ON files(document_kind)
  WHERE document_kind IS NOT NULL;

CREATE TABLE documents (
    id      INTEGER PRIMARY KEY,
    file_id INTEGER NOT NULL REFERENCES files(id) ON DELETE CASCADE,
    kind    TEXT    NOT NULL,
    key     TEXT    NOT NULL,
    value   TEXT    NOT NULL,
    line    INTEGER NOT NULL
);
CREATE INDEX idx_documents_file ON documents(file_id);
CREATE INDEX idx_documents_kind ON documents(kind);
CREATE INDEX idx_documents_key  ON documents(key);

CREATE VIRTUAL TABLE documents_fts USING fts5(
    key, value,
    content='documents',
    content_rowid='id',
    tokenize='trigram'
);
-- triggers mirroring symbols_fts (ai/ad/au)
```

## Parser interface

`internal/parser/document.go` (new):

```go
type DocumentEntry struct {
    Key   string
    Value string
    Line  int
}

type DocumentResult struct {
    Path        string
    Kind        string
    Entries     []DocumentEntry
    ContentHash []byte
}

type DocumentParser interface {
    Kind() string
    Supports(path string) bool
    Parse(ctx context.Context, path string, content []byte) (DocumentResult, error)
}
```

Parallel to `parser.Parser`. No `Language()`.

`internal/parser/document/registry.go` (new) — same shape as
`parser/registry.go`: `Register`, `ForPath`, `Kinds()`.

## i18n JSON parser

`internal/parser/document/i18n_json.go` (new). Detection:

```go
func (p *I18NJSON) Supports(path string) bool {
    s := filepath.ToSlash(path)
    return strings.HasSuffix(s, ".json") &&
        (strings.Contains(s, "/locales/") ||
            strings.Contains(s, "/i18n/"))
}
```

Key flattening: nested objects join with `.`, arrays use numeric
indices. Only string values become entries; objects and arrays are
flattened, numbers/bools/null are skipped (i18n values are always
strings in practice).

```
{
  "topbar": {
    "nav": {
      "back": "Go back",
      "items": ["one", "two"]
    }
  }
}
```

becomes:

```
topbar.nav.back     "Go back"
topbar.nav.items.0  "one"
topbar.nav.items.1  "two"
```

Line tracking: use `encoding/json` with a `json.Decoder` that
reports offsets, then translate offset → line. (Alternative: a
hand-written scanner. Standard-library decoder is simpler; revisit
if perf becomes an issue on large locale files.)

## Pipeline wiring

`internal/pipeline/pipeline.go` currently dispatches to the symbol
`Parser` registry. Add a parallel `DocumentRegistry` field. For each
file walked:

1. Ask the symbol registry: any parser claims it? → existing code path.
2. Ask the document registry: any parser claims it? → run it, write
   `documents` rows + set `files.document_kind`.
3. A file matched by neither is skipped (existing behaviour).

If a file matches both (e.g. a future `code_with_keys` mixed file),
both run. v3.3 has no such overlap; the registries are independent.

## Index write path

`internal/index/upsert.go` (or wherever files/symbols are written) —
new function `UpsertDocumentEntries(ctx, fileID, kind, entries)`.
Same delete-then-insert pattern used for symbols. Skip when entries
are unchanged (compare against the file's existing entries by
content_hash on the parser result — same machinery as parse_hash
for symbols).

- `task check` passes (vet + race tests, no symbol-graph regressions).

  verify the flat-key output.

  is registered in `files`).

  modifies a shipped file, runs idempotently on an existing DB.
