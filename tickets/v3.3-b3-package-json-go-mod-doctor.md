# B3 ‚Äî package.json + go.mod parsers + doctor coverage

**Priority:** P1 for v3.3 ‚Äî completes the v3.3 document inventory
**Depends on:** [v3.3-b1-schema-and-i18n-json-parser.md](v3.3-b1-schema-and-i18n-json-parser.md)
**Plan reference:** `~/.claude/plans/9-v3.3-documents-surface.md`

## Goal

Two more document kinds plus the doctor coverage check. With B1's
infrastructure in place this is mostly additive: each parser is one
small file implementing `DocumentParser`.

## package.json parser

`internal/parser/document/package_json.go` (new).

Detection: `filepath.Base(path) == "package.json"`.

Emit one entry per dependency across `dependencies`,
`devDependencies`, `peerDependencies`, `optionalDependencies`.

```json
{
  "dependencies": {
    "react": "^18.0.0",
    "@codesphere/utils-common": "workspace:*"
  }
}
```

becomes:

```
react                            "^18.0.0"
@codesphere/utils-common         "workspace:*"
```

Kind: `package_json_deps`. Don't emit the section name ‚Äî the value
already encodes the range, and consumers usually search by name. If
section disambiguation becomes useful later (e.g. "is this a
devDependency?"), revisit.

## go.mod parser

`internal/parser/document/go_mod.go` (new).

Detection: `filepath.Base(path) == "go.mod"`.

Parse with `golang.org/x/mod/modfile` (already a transitive dep
via Go's tooling, or add it ‚Äî check go.sum). Emit one entry per
`require` block entry:

```
require (
    github.com/foo/bar v1.2.3
    github.com/baz/quux v0.0.0-20240101000000-deadbeef
)
```

becomes:

```
github.com/foo/bar         "v1.2.3"
github.com/baz/quux        "v0.0.0-20240101000000-deadbeef"
```

Kind: `go_mod_requires`. `replace` and `exclude` directives are out
of scope for v3.3 (rare in agent-facing workflows).

## Doctor check

`internal/doctor/checks.go` ‚Äî new check:

```go
type DocumentCoverage struct {
    Kind  string
    Count int
}
```

Surface in `myco doctor` output:

```
documents_indexed       i18n_json:42 package_json_deps:8 go_mod_requires:1
```

Failure mode: a kind that's enabled in the parser registry but has
zero entries in the DB. Same shape as v3.1's
`projects_configured_but_empty`. Hint copy:

> "kind X is registered but no files matched; check that files like
> `locales/*.json` / `package.json` / `go.mod` exist and aren't
> excluded by your config."

For v3.3 all three kinds are always registered (no per-repo
enable/disable), so the warning only fires for a genuine empty
state (e.g. a Go-only repo correctly has zero `i18n_json` and that
should NOT warn ‚Äî gate the warning on whether at least one
candidate file *exists* in the workspace, not just whether the
parser is registered). Simpler: only warn when document files *do*
exist on disk but produce zero entries ‚Äî that's a real bug. Skip
the "kind always registered" case.

## Stats integration

`internal/query/query.go` `Stats` ‚Äî add per-kind document counts to
the existing stats payload. Single SQL:

```sql
SELECT kind, COUNT(*) FROM documents GROUP BY kind
```

Add `DocumentsByKind map[string]int` to the `Stats` struct.

## Acceptance criteria

- `task check` passes.
- Indexing the documents fixture with all three file kinds produces
  the expected entries per kind.
- `myco doctor` output includes `documents_indexed` with non-zero
  counts; the empty-state warning only fires when document files
  exist but produce zero entries.
- `myco stats` payload includes `documents_by_kind`.

## Out of scope

- `replace` / `exclude` directives in `go.mod`.
- Workspaces in `package.json` (`workspaces:` field) ‚Äî that's a
  packaging structure thing, the per-package deps already cover it.
- Lockfiles (`package-lock.json`, `yarn.lock`, `go.sum`) ‚Äî these
  duplicate what's in the canonical files at much higher cost.
- A `find_document_value` query (substring match against values).
  `search_lexical` already does this with no new code.
